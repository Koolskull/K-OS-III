# Why cryptography matters before you touch a wallet

*This lesson is a stub. See `EDUCATION.md` for the project's curriculum and `docs/SEED_ISSUES.md` for the issue to flesh this out.*

This track is a prerequisite for the `privacy/`, `bitcoin/`, `monero/`, and (informally) the K-Wallet portion of the user experience. It exists because most developers and users meet cryptography for the first time when they need it to work, which is the worst possible time to learn it. By the end of this track you should know what a hash is, what a signature is, what a key derivation function is, why seed phrases look like they do, and where each of these shows up inside K-OS.

The track is concept-first. Math is introduced only when it earns its place; you do not need to know how a SHA-256 round works to use one correctly.
