# Who this track is for

*This lesson is a stub. See `EDUCATION.md` for the project's curriculum and `docs/SEED_ISSUES.md` for the issue to flesh this out.*

This track follows the `ethereum-dev/` track and assumes the reader has already shipped at least one Solidity contract end-to-end — written it, tested it, deployed it, and watched it run. Where the previous track is breadth-first, this one is depth-first.

Topics covered (full lesson outlines in `track.yaml`):

- Storage layout and slot packing
- Gas optimization that's actually worth doing
- Design patterns: proxies, access control, factory, registry, escrow
- Security patterns: pull payments, checks-effects-interactions, reentrancy guards, pause switches
- What an audit-ready contract looks like
